想要进行本文的步骤，最好需要有：

- 一台小米 11 (对于 Pro, Ultra 和 S，本文也许并不适用)
- 一台能够连接互联网的电脑
- 一根连接稳定的 USB Type-A to Type-C 数据线(其实送的充电线就行)
- 一张能够联网的 SIM 卡
- 你或许也需要一个代理用来访问被神必东方力量屏蔽的网站

## 环境配置

USB 驱动

[ADB 工具包 (Android SDK Platform-Tools)](https://developer.android.com/studio/releases/platform-tools)

这些在网络上已有许多教程，想要验证也非常简单，在配置好环境变量后打开终端或命令提示符输入以下命令

```shell
adb reboot bootloader
```

![adb reboot bootloader](https://chitangcos.zyglq.cn/images/mi11/adb%20reboot.png)_忘记写 `bootloader` 了~~，问题不大~~_

如果配置没有问题，手机上应该会弹出一个类似这样索要权限的窗口

![ADB 授权](https://chitangcos.zyglq.cn/images/mi11/adb-auth.webp)_~~没在 MIUI 用类原生糊弄一下~~_

推荐勾选 <u>一律允许使用这台计算机进行调试</u>

在点击允许后，手机应该会重启，然后进入 fastboot 模式

![fastboot](https://chitangcos.zyglq.cn/images/mi11/fastboot.webp)

这时，再执行

```shell
fastboot reboot
```

![fastboot reboot](https://chitangcos.zyglq.cn/images/mi11/fastboot%20reboot.png)

如果配置没问题，现在手机应该会正常重启并进入系统

## 解锁

解锁 Bootloader 是玩机的前提

### 绑定帐号

<div class="success">

> 如果之前绑定过帐号，无需再次进行本步骤

</div>

前往 <u>设置</u> <u>我的设备</u> <u>全部参数</u>

快速点击五次 <u>MIUI 版本</u>，启用开发者选项

回到设置主页，前往 <u>其他设置</u> <u>开发者选项</u>

启用 <u>OEM 解锁</u>

在 <u>设备解锁状态</u> 中点击 <u>绑定帐号与手机</u>

绑定成功后，等待 168 小时(也就是整整一周)

### 备份数据

使用官方工具解锁 Bootloader 会导致丢失数据，因此在正式解锁前需要对数据进行备份 ~~当然如果你恰好想恢复出厂设置也没人拦着你~~

使用 MIUI 自带的备份可能是在没有 Root 的情况下最好的选择

前往 <u>设置</u> <u>我的设备</u> <u>备份与恢复</u>

如果有 OTG 和足够存储空间的 U 盘，可以选择 <u>U盘备份恢复</u>，否则，选择 <u>手机备份恢复</u>

新建备份，选择自己需要备份的内容，开始备份

至于图片、通讯录、短信等，若数量不多使用 MIUI 云备份足矣

### 使用解锁工具

该步骤需要一台 Windows 设备

将手机连接到电脑，执行该指令以进入 fastboot

```shell
adb reboot bootloader
```

下载 [MIUI 解锁工具](https://www.miui.com/unlock/download.html)

解压后运行 `miflash_unlock.exe`，根据提示登录**与手机绑定的同一个帐号**并解锁手机

在经过一段时间后，手机会自动重启，这时第一屏上方会增加一个解锁的锁头

![锁头](https://chitangcos.zyglq.cn/images/mi11/unlocked.webp)

## 第三方 Recovery

本段中的三种 Recovery 可以根据需要自行选择

### TWRP

前往 [TWRP 官网](https://dl.twrp.me/venus/)下载最新的 .img 文件

重启至 Fastboot

```shell
adb reboot bootloader
```

然后，临时启动至 TWRP

```shell
fastboot boot twrp-xxxxx.img
```

将 `twrp-xxxxx.img` 替换为你所下载的文件名

![Boot TWRP](https://chitangcos.zyglq.cn/images/mi11/boot-twrp.png)

待手机进入 TWRP Recovery 后，进入 <u>Advanced</u> 菜单

![TWRP Advanced](https://chitangcos.zyglq.cn/images/mi11/twrp-advanced.webp)

然后，选择 <u>Flash Current TWRP</u>

![TWRP Flash](https://chitangcos.zyglq.cn/images/mi11/twrp-flash.webp)

滑动滑块以进行确认

在标题变为 <u>Done flashing TWRP</u> 后，点击 <u>Reboot System</u>，TWRP Recovery 就已经成功安装

![TWRP Done](https://chitangcos.zyglq.cn/images/mi11/twrp-done.webp)

### Pixel Experience Recovery

只有在安装 Pixel Experience 时需要本 Recovery

前往 [Pixel Experience 下载页](https://download.pixelexperience.org/venus)，选择所需的版本，点击 <u>Download recovery image</u>

![下载](https://chitangcos.zyglq.cn/images/mi11/download-pe.png)

使用 Pixel Experience Recovery 还需要[在这里](https://gitlab.pixelexperience.org/android/vendor-blobs/wiki_blobs_venus/-/raw/twelve/android-12/vendor_boot.img)下载一个 `vender_boot` 镜像

重启至 Fastboot

```shell
adb reboot bootloader
```

在进入 Fastboot 模式后，刷入 `vendor_boot`

```shell
fastboot flash vendor_boot vendor_boot.img
```

然后，刷入 Pixel Experience Recovery

```shell
fastboot flash boot PixelExperience_xxxxxx.img
```

将 `PixelExperience_xxxxxxx.img` 替换为你所下载的文件名

![Flash Pixel Experience Recovery](https://chitangcos.zyglq.cn/images/mi11/flash_pe_rec.png)

在两条命令均 Finish 后即可重启

```shell
fastboot reboot
```

## Magisk

### 刷入

在 [Magisk Release 页面](https://github.com/topjohnwu/Magisk/releases/)下载最新的 apk

重启至你所使用的第三方 Recovery，本段以 Pixel Experience Recovery 为例

```shell
adb reboot recovery
```

选择 <u>Apply update</u> <u>Apply from ADB</u> 以启动 ADB Sideload 模式

回到电脑，使用这个命令来刷入 Magisk

```shell
adb sideload Magisk-xxx.apk
```

将 `Magisk-xxx.apk` 替换为你所下载的文件名

完成后，在 Recovery 中重启手机至系统

最后，安装 Magisk Manager

```shell
adb install Magisk-xxx.apk
```

![Flash Magisk](https://chitangcos.zyglq.cn/images/mi11/flash-magisk.png)

现在，打开手机上新增的 Magisk 应用，若下方的 <u>超级用户</u> 和 <u>模块</u> 菜单能够点击，就代表已经成功获取了 Root 权限

### 设置

打开 Magisk，前往右上角的设置

![Magisk Settings](https://chitangcos.zyglq.cn/images/mi11/magisk-settings.png)

从上往下来简单讲解一下部分设置的用途及推荐设置

| 名称           | 作用                                                                                   | 推荐值  |
| ------------ | ------------------------------------------------------------------------------------ | ---- |
| 安全 DNS (DoH) | 关于 DoH 的作用可以查看[维基百科页面](https://zh.wikipedia.org/wiki/DNS_over_HTTPS)，启用可以防止部分 DNS 污染 | 启用   |
| 隐藏 Magisk 应用 | 生成一个随机包名的 Magisk App 并安装，用于防止其他 App 检测 Magisk 包名                                     | 根据需求 |
| Zygisk       | 我不知道                                                                                 | 启用   |
| 遵守排除列表       | 使用 Magisk 内置的隐藏功能，但启用会导致后续安装的 Shamiko 隐藏模块失效                                         | 禁用   |
| 配置排除列表       | 配置需要对其隐藏 Magisk 的应用                                                                  | 按需选择 |
| 生物识别验证       | 在其他 App 申请 Root 权限时使用系统生物识别方式进行身份验证                                                  | 启用   |
